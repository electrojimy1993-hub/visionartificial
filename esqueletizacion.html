Esqueletización con Google MediaPipe

Google MediaPipe es un ecosistema de herramientas de Machine Learning multiplataforma de código abierto, diseñado para procesar flujos de datos multimedia en tiempo real. A diferencia de las redes neuronales pesadas, los pipelines de MediaPipe están optimizados para correr en hardware modesto (como CPUs de computadoras convencionales o smartphones), logrando tasas superiores a los 30 FPS.

1. Hand Landmark Detection (Manos)

Esta solución detecta la palma de la mano y extrae un total de 21 coordenadas clave (landmarks) en 3D por cada mano identificada en la imagen. La coordenada $Z$ estima la profundidad del punto respecto a la articulación de la muñeca.

Estructura Topológica Clave:

  • 0: Muñeca (WRIST)
  • 1-4: Pulgar (THUMB_CMC a THUMB_TIP)
  • 5-8: Índice (INDEX_FINGER_MCP a INDEX_FINGER_TIP)
  • 9-12: Medio (MIDDLE_FINGER_MCP a MIDDLE_FINGER_TIP)
  • 13-16: Anular (RING_FINGER_MCP a RING_FINGER_TIP)
  • 17-20: Meñique (PINKY_MCP a PINKY_TIP)

2. Pose Landmark Detection (Cuerpo Completo)

Este modelo infiere la pose tridimensional de un cuerpo completo a partir de una única imagen RGB, entregando un total de 33 marcas anatómicas clave. Adicionalmente, provee un factor de visibilidad ($visibility$) que estima la probabilidad de que el punto no esté obstruido en el plano.

Puntos Clave de Control Corporal:

  • 0-10: Rasgos Faciales (Ojos, Orejas, Nariz, Boca)
  • 11-12: Hombros Izquierdo (11) y Derecho (12)
  • 13-14: Codos Izquierdo (13) y Derecho (14)
  • 15-16: Muñecas Izquierda (15) y Derecha (16)
  • 23-24: Caderas Izquierda (23) y Derecha (24)
  • 25-28: Rodillas (25-26) y Tobillos (27-28)

Pipeline Esencial de Programación con MediaPipe

Para interactuar con la cámara usando Python, creamos un ciclo infinito, transformamos la paleta de colores de BGR a RGB (ya que MediaPipe opera nativamente bajo esta norma), alimentamos la red y luego pintamos los puntos de referencia en la pantalla utilizando el módulo drawing_utils de la librería. 

import cv2
import mediapipe as mp

# 1. Instanciar utilidades de dibujo y modelos
mp_drawing = mp.solutions.drawing_utils
mp_pose = mp.solutions.pose  # Cambiar a mp.solutions.hands para manos

cap = cv2.VideoCapture(0)

# 2. Configurar el detector
with mp_pose.Pose(min_detection_confidence=0.5, min_tracking_confidence=0.5) as pose:
    while cap.isOpened():
        ret, frame = cap.read()
        if not ret: break
        
        # Transformar el frame de BGR a RGB
        image_rgb = cv2.cvtColor(frame, cv2.COLOR_BGR2RGB)
        
        # Procesar imagen para detectar marcas
        results = pose.process(image_rgb)
        
        # Dibujar esqueleto si hay detecciones
        if results.pose_landmarks:
            mp_drawing.draw_landmarks(
                frame, 
                results.pose_landmarks, 
                mp_pose.POSE_CONNECTIONS
            )
            
        cv2.imshow('MediaPipe Tracking', frame)
        if cv2.waitKey(10) & 0xFF == ord('q'):
            break

cap.release()
cv2.destroyAllWindows()

Integración de Visión y Control de Motores

Caso Industrial: Manipulación de Brazo Robótico (Visual Servoing)

En las líneas de ensamble automatizado modernas, los cobots (robots colaborativos) interactúan y replican movimientos de operadores humanos en tiempo real. Esto disminuye la fatiga en tareas repetitivas o manipulación de materiales peligrosos.

Planteamiento del Reto

Diseña e implementa un script en Python que capture el esqueleto completo de un operador por webcam, filtre y aísle exclusivamente el brazo derecho (hombro, codo y muñeca) mediante el modelo mp.solutions.pose. El algoritmo debe realizar el cálculo matemático del ángulo tridimensional en el codo del operador y comunicarlo vía puerto Serial a un microcontrolador (Arduino) para mover un servomotor del brazo mecánico en tiempo real.

Lógica Vectorial para Cálculo de Ángulos:

Para obtener el ángulo de articulación del codo $B$, definimos los vectores entre el hombro $A(x_1, y_1)$, el codo $B(x_2, y_2)$ y la muñeca $C(x_3, y_3)$.

$$\vec{v_1} = (x_1 - x_2, \ y_1 - y_2)$$ $$\vec{v_2} = (x_3 - x_2, \ y_3 - y_2)$$ $$\theta = \arccos\left(\frac{\vec{v_1} \cdot \vec{v_2}}{\|\vec{v_1}\| \|\vec{v_2}\|}\right) \times \frac{180}{\pi}$$

Pista de OpenCV: Una vez obtenido el ángulo en grados (0° a 180°), es recomendable mapearlo al rango de operación física de tu servomotor y limitar las fluctuaciones de ruido aplicando un promedio móvil o filtro de paso bajo por software.

Requerimientos de la Entrega:

  1. Bucle de Cámara: El script debe encender tu webcam usando OpenCV y procesar los fotogramas dinámicamente con MediaPipe.
  2. Aislamiento y Trazado: Mostrar en pantalla el video original pero dibujando únicamente las marcas del hombro derecho (12), codo derecho (14) y muñeca derecha (16). Debes trazar líneas rectas que los unan.
  3. Cálculo Angular: Calcular de forma dinámica el ángulo del codo en base a la matemática vectorial y proyectar el valor textual en pantalla con cv2.putText.
  4. Comunicación Serial: Transmitir la variable angular a través de PySerial al puerto donde esté conectado tu microcontrolador Arduino con una estructura de delimitador clara.

Nota Académica: No se proporciona el código integrado para este reto. Deberás investigar cómo estructurar la biblioteca PySerial, el cálculo de arco coseno usando numpy.arctan2 o numpy.arccos, y escribir el bosquejo (.ino) que lea los comandos desde el búfer del puerto serie de la placa de desarrollo.
